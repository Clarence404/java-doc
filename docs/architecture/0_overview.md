# 系统架构总览

> 本模块聚焦 Java 后端系统架构设计，覆盖架构演进、核心模式、DDD、访问控制、数据策略等主题。

---

## 一、架构演进路线

| 阶段 | 特征 | 适用规模 | 核心问题 |
|------|------|---------|---------|
| **单体架构** | 所有功能打包一个应用 | < 10人 | 快速交付，MVP 验证 |
| **SOA** | 粗粒度服务 + ESB 总线 | 10~50人 | 跨部门服务复用 |
| **微服务** | 细粒度服务，独立部署 | 50人以上 | 独立扩容，技术栈灵活 |
| **云原生** | 容器化 + 服务网格 + Serverless | 大型组织 | 弹性伸缩，零停机交付 |

演进规律：**先单体，后拆分**。不要在业务不清晰时过早拆分微服务，过细的服务边界会带来分布式事务、链路复杂、运维成本三重负担。

---

## 二、架构师核心职责

- **技术选型**：评估框架、中间件、云服务的成本与风险，输出选型报告
- **系统分解**：按 DDD 限界上下文划定服务边界，识别核心域/支撑域/通用域
- **非功能设计**：可用性（SLA）、性能（P99）、安全（等保）、成本
- **架构决策记录（ADR）**：记录"为什么这样设计"，供未来参考
- **技术债管理**：识别、量化、计划还债，防止带病上线

## 三、技术选型决策框架

| 维度 | 说明 | 权重参考 |
|------|------|---------|
| **成熟度** | 社区活跃、版本稳定、有生产案例 | 高 |
| **团队熟悉度** | 现有团队学习成本 | 高 |
| **性能** | 吞吐量、延迟、资源消耗 | 中 |
| **运维成本** | 部署复杂度、故障排查、监控接入 | 中 |
| **生态兼容** | 与现有技术栈的集成成本 | 中 |
| **许可证** | Apache/MIT/GPL 商业风险 | 低 |

常见选型对比：

| 组件 | 推荐（国内） | 备选 | 说明 |
|------|------------|------|------|
| 注册/配置中心 | Nacos | Apollo + Eureka | Nacos 一体化，轻量首选 |
| 网关 | Spring Cloud Gateway | APISIX | Gateway 原生响应式，APISIX 插件生态丰富 |
| 熔断限流 | Sentinel | Resilience4j | Sentinel 有可视化控制台 |
| 消息队列 | RocketMQ / Kafka | RabbitMQ | 事务消息用 RocketMQ，日志/流处理用 Kafka |
| 缓存 | Redis | Caffeine（本地） | 二级缓存：Caffeine + Redis |
| 链路追踪 | SkyWalking | Jaeger | SkyWalking Java 探针无侵入 |
