## 一、PostgreSQL 9.6（2016 年 9 月）

### 1、并行查询（Parallel Query）初步支持

首次引入并行查询执行，支持以下操作并行化：
- 顺序扫描（Sequential Scan）
- Hash Join / Nested Loop
- 聚合函数（部分聚合算子）

```sql
-- 控制并行度
SET max_parallel_workers_per_gather = 4;

-- 查看执行计划是否使用了并行
EXPLAIN ANALYZE SELECT sum(amount) FROM orders;
-- 输出中出现 Gather 节点即表示并行执行
```

### 2、同步复制支持多备库

可指定多个备库参与同步提交，quorum 式提交：

```ini
# postgresql.conf
synchronous_standby_names = 'FIRST 1 (replica1, replica2)'  # 至少 1 个确认
```

### 3、pg_stat_activity 等待事件可视化

新增 `wait_event_type` / `wait_event` 字段，可精确看到会话在等什么锁或资源：

```sql
SELECT pid, state, wait_event_type, wait_event, query
FROM pg_stat_activity
WHERE state != 'idle';
```

### 4、postgres_fdw 聚合下推

外部表（Foreign Table）的聚合计算可下推到远端服务器执行，减少数据传输量。

## 二、PostgreSQL 10（2017 年 10 月）

### 1、声明式分区（Declarative Partitioning）

取代基于继承表的手工分区方案，语法简洁：

```sql
-- 范围分区
CREATE TABLE orders (id BIGINT, created_at DATE, amount NUMERIC)
PARTITION BY RANGE (created_at);

CREATE TABLE orders_2024 PARTITION OF orders
  FOR VALUES FROM ('2024-01-01') TO ('2025-01-01');

CREATE TABLE orders_2025 PARTITION OF orders
  FOR VALUES FROM ('2025-01-01') TO ('2026-01-01');

-- 列表分区
CREATE TABLE logs PARTITION BY LIST (region);
CREATE TABLE logs_cn PARTITION OF logs FOR VALUES IN ('cn');
CREATE TABLE logs_us PARTITION OF logs FOR VALUES IN ('us');
```

### 2、逻辑复制（Logical Replication）

基于发布/订阅模型，可选择性地复制指定表，支持跨版本复制：

```sql
-- 发布端（源库）
CREATE PUBLICATION pub_orders FOR TABLE orders, order_items;

-- 订阅端（目标库）
CREATE SUBSCRIPTION sub_orders
  CONNECTION 'host=source_db dbname=prod user=replicator'
  PUBLICATION pub_orders;
```

### 3、标识列（Identity Columns）

符合 SQL 标准的自增列，替代 `SERIAL`：

```sql
-- GENERATED ALWAYS AS IDENTITY：不允许手动插入值
-- GENERATED BY DEFAULT AS IDENTITY：允许手动插入值
CREATE TABLE users (
  id   BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  name TEXT
);

INSERT INTO users (name) VALUES ('Alice');  -- id 自动生成
```

### 4、Hash 索引崩溃安全

10 之前 Hash 索引不写 WAL，崩溃后需重建。10 起 Hash 索引完全崩溃安全，可放心使用。

### 5、SCRAM-SHA-256 认证

新增更安全的密码认证方式，替代旧版 md5：

```ini
# pg_hba.conf
host  all  all  0.0.0.0/0  scram-sha-256
```

## 三、PostgreSQL 11（2018 年 10 月）

### 1、存储过程（CREATE PROCEDURE）

区别于函数（Function），存储过程支持在过程内**控制事务**：

```sql
CREATE PROCEDURE transfer(from_id INT, to_id INT, amount NUMERIC)
LANGUAGE plpgsql AS $$
BEGIN
  UPDATE accounts SET balance = balance - amount WHERE id = from_id;
  UPDATE accounts SET balance = balance + amount WHERE id = to_id;
  COMMIT;  -- 存储过程内可提交/回滚
END;
$$;

CALL transfer(1, 2, 100.00);
```

### 2、JIT 即时编译（Just-in-Time Compilation）

对表达式求值进行 LLVM JIT 编译，大幅加速复杂分析查询（CPU 密集型）：

```sql
-- 开关控制
SET jit = on;          -- 开启（11 引入，12 默认开启）
SET jit_above_cost = 100000;  -- 代价超过此值才触发 JIT
```

### 3、覆盖索引（INCLUDE 列）

索引可包含非键列，避免回表（类似 MySQL 覆盖索引）：

```sql
-- name 是索引键，email 是附加列（不参与排序，但查询时可避免回表）
CREATE INDEX idx_users_name ON users (name) INCLUDE (email);

-- 以下查询不需要访问主表（Index Only Scan）
SELECT name, email FROM users WHERE name = 'Alice';
```

### 4、Hash 分区

10 只有 RANGE 和 LIST，11 新增 HASH 分区：

```sql
CREATE TABLE users PARTITION BY HASH (user_id);
CREATE TABLE users_0 PARTITION OF users FOR VALUES WITH (MODULUS 4, REMAINDER 0);
CREATE TABLE users_1 PARTITION OF users FOR VALUES WITH (MODULUS 4, REMAINDER 1);
CREATE TABLE users_2 PARTITION OF users FOR VALUES WITH (MODULUS 4, REMAINDER 2);
CREATE TABLE users_3 PARTITION OF users FOR VALUES WITH (MODULUS 4, REMAINDER 3);
```

### 5、分区表增强

- 分区表上可直接创建主键、外键、唯一索引
- 新增 DEFAULT 分区（兜底分区）

## 四、PostgreSQL 12（2019 年 10 月）

### 1、生成列（Generated Columns）

根据其他列的表达式自动计算，物理存储（STORED）：

```sql
CREATE TABLE products (
  price    NUMERIC,
  qty      INT,
  total    NUMERIC GENERATED ALWAYS AS (price * qty) STORED
);

INSERT INTO products (price, qty) VALUES (9.99, 5);
SELECT total FROM products;  -- 自动计算为 49.95
```

### 2、CTE 优化屏障取消（默认内联）

12 之前 CTE 是"优化屏障"（总是 Materialize），12 起默认内联优化，性能更好：

```sql
-- 12 之前：WITH 子查询总是独立执行，无法与外部查询合并优化
-- 12 起：优化器默认内联，必要时手动加 MATERIALIZED 恢复旧行为
WITH recent AS MATERIALIZED (    -- 强制物化，保留旧行为
  SELECT * FROM orders WHERE created_at > now() - interval '7d'
)
SELECT * FROM recent WHERE amount > 100;
```

### 3、REINDEX CONCURRENTLY

重建索引时不阻塞读写（以前 REINDEX 会锁表）：

```sql
REINDEX INDEX CONCURRENTLY idx_orders_user_id;
REINDEX TABLE CONCURRENTLY orders;
```

### 4、B-tree 索引优化

12 大幅重写 B-tree 索引实现，减少多列索引膨胀，非唯一索引空间可节省 **30~40%**。

### 5、多列 MCV 统计

优化器可以对多列联合分布建立统计，解决相关列查询估算不准的问题：

```sql
-- 对 (status, region) 组合建立 MCV 统计
CREATE STATISTICS stat_status_region ON status, region FROM orders;
ANALYZE orders;
-- 之后优化器对 WHERE status='paid' AND region='cn' 的行数估算更准确
```

### 6、pg_checksums

对已有集群启用/禁用数据页校验和（需短暂停机）：

```bash
pg_checksums --enable -D /data/pgdata
```

## 五、PostgreSQL 13（2020 年 9 月）

### 1、B-tree 索引去重（Deduplication）

对非唯一索引中的重复键值进行去重压缩存储，**索引体积可减少 30~70%**：

```sql
-- 自动生效，无需手动配置
-- 对 status、region 等低区分度列建的索引效果最显著
CREATE INDEX idx_orders_status ON orders (status);
-- 13 起该索引自动去重，体积远小于 12
```

### 2、FETCH FIRST ... WITH TIES

返回与第 N 行排序值相同的所有行（SQL 标准语法）：

```sql
-- 取排名前 3，包括并列第 3 的所有行
SELECT name, score
FROM leaderboard
ORDER BY score DESC
FETCH FIRST 3 ROWS WITH TIES;
```

### 3、大事务流式逻辑复制

逻辑复制订阅端可在事务**提交前**就开始流式接收并应用，减少大事务复制延迟。

### 4、pg_verifybackup

验证基础备份的完整性（校验文件 + WAL）：

```bash
pg_verifybackup /backup/base
```

### 5、分区表部分聚合并行优化

分区表的聚合查询支持分区级别并行执行，OLAP 场景性能明显提升。

## 六、PostgreSQL 14（2021 年 9 月）

### 1、多范围类型（Multirange）

单个值可以表示多个不连续的范围：

```sql
-- 普通范围类型（13 及之前）
SELECT '[2024-01-01, 2024-01-31]'::daterange;

-- 多范围类型（14 新增）
SELECT '{[2024-01-01,2024-01-15],[2024-02-01,2024-02-15]}'::datemultirange;

-- 判断某日期是否在任意一段范围内
SELECT '2024-01-10'::date <@ '{[2024-01-01,2024-01-15],[2024-02-01,2024-02-15]}'::datemultirange;
-- true
```

### 2、JSONB 下标访问

```sql
-- 14 之前：只能用 -> 操作符
SELECT data->'address'->'city' FROM users WHERE id = 1;

-- 14 起：支持下标语法，更直观
SELECT data['address']['city'] FROM users WHERE id = 1;

-- 也可用于 UPDATE
UPDATE users SET data['address']['city'] = '"Shanghai"' WHERE id = 1;
```

### 3、存储过程 OUT 参数

```sql
CREATE PROCEDURE get_order_stats(
  IN  user_id  INT,
  OUT total    NUMERIC,
  OUT cnt      INT
) LANGUAGE sql AS $$
  SELECT SUM(amount), COUNT(*) FROM orders WHERE orders.user_id = $1;
$$;

CALL get_order_stats(123, NULL, NULL);
```

### 4、libpq 管道模式（Pipeline Mode）

客户端可批量发送多条查询而无需等待每条响应，显著降低高延迟网络下的往返时间开销：

```c
// libpq C API（Java 通过 JDBC 间接受益）
PQenterPipelineMode(conn);
PQsendQuery(conn, "INSERT ...");
PQsendQuery(conn, "INSERT ...");
PQpipelineSync(conn);
```

### 5、B-tree 自底向上删除

对频繁 UPDATE 的列上的索引，主动清理过期版本，**减少索引膨胀**，降低 VACUUM 压力。

### 6、SQL/JSON 路径函数（部分）

新增 `jsonb_path_query_array`、改进 `jsonb_path_*` 系列函数的行为。

## 七、PostgreSQL 15（2022 年 10 月）

### 1、MERGE 语句（SQL 标准）

```sql
MERGE INTO inventory AS t
USING incoming AS s ON t.sku = s.sku
WHEN MATCHED AND s.qty > 0 THEN
  UPDATE SET qty = t.qty + s.qty, updated_at = now()
WHEN MATCHED AND s.qty = 0 THEN
  DELETE
WHEN NOT MATCHED THEN
  INSERT (sku, qty) VALUES (s.sku, s.qty);
```

### 2、逻辑复制行/列过滤

发布时可指定行过滤条件和列白名单，只复制部分数据：

```sql
-- 只发布 status='active' 的行，只复制 id、name、email 三列
CREATE PUBLICATION pub_active_users FOR TABLE users (id, name, email)
  WHERE (status = 'active');
```

### 3、排序性能提升

内存排序算法重写，归并排序（Merge Sort）性能提升显著，大数据集排序速度约提升 **25%**。

### 4、WAL 压缩支持 lz4 / zstd

```ini
# postgresql.conf
wal_compression = lz4     # 或 zstd，压缩率更高，写入 WAL 体积更小
```

### 5、结构化 JSON 日志

```ini
log_destination = 'jsonlog'    # 输出 JSON 格式日志，便于接入 ELK 等日志系统
```

### 6、权限管理增强

新增 `pg_read_all_data` / `pg_write_all_data` 预定义角色：

```sql
-- 授予只读所有表的权限（无需逐表 GRANT）
GRANT pg_read_all_data TO readonly_user;
```

## 八、PostgreSQL 16（2023 年 9 月）

### 1、并行查询增强

FULL JOIN 和 RIGHT JOIN 支持并行执行，带 `DISTINCT` / `ORDER BY` 的聚合、`SELECT DISTINCT` 可利用增量排序：

```sql
SET max_parallel_workers_per_gather = 4;
EXPLAIN ANALYZE
SELECT a.*, b.* FROM large_a a FULL JOIN large_b b ON a.id = b.id;
-- 执行计划出现 Parallel Hash Full Join
```

### 2、SIMD CPU 加速

引入 x86（AVX-2）和 ARM（NEON）SIMD 指令，加速 ASCII/JSON 字符串处理、数组搜索等操作。

### 3、COPY 性能大幅提升

批量导入在单并发和多并发场景部分测试提升达 **300%**。

### 4、从备库发起逻辑复制

Standby 可作为逻辑复制发布者，减轻主库 WAL 解码压力：

```sql
-- 在备库上创建发布（主库需 wal_level=logical）
CREATE PUBLICATION pub_orders FOR TABLE orders;
```

### 5、SQL/JSON 构造函数

```sql
SELECT JSON_ARRAY(1, 'two', NULL);                    -- [1, "two", null]
SELECT JSON_ARRAYAGG(name) FROM tags;
SELECT JSON_OBJECT('id': 1, 'name': 'Alice');
SELECT '{"a":1}' IS JSON WITH UNIQUE KEYS;            -- 检查 key 唯一性
```

### 6、pg_stat_io 视图

```sql
SELECT backend_type, object, context, reads, hits, evictions
FROM pg_stat_io
ORDER BY reads DESC;
```

### 7、角色继承细粒度控制

```sql
GRANT analyst TO alice WITH INHERIT TRUE;
GRANT dba     TO alice WITH INHERIT FALSE;   -- 需手动 SET ROLE dba 才激活
```

## 九、PostgreSQL 17（2024 年 9 月）

### 1、增量备份

```bash
pg_basebackup -D /backup/base --checkpoint=fast -Ft -z
pg_basebackup -D /backup/inc \
  --incremental=/backup/base/backup_manifest \
  --checkpoint=fast -Ft -z
pg_combinebackup /backup/base /backup/inc -o /restore/full
```

### 2、JSON_TABLE 及 JSON 查询函数

```sql
SELECT jt.* FROM orders,
  JSON_TABLE(items, '$[*]' COLUMNS (
    product_id INT    PATH '$.id',
    qty        INT    PATH '$.qty'
  )) AS jt;

SELECT JSON_EXISTS(data, '$.address.city');
SELECT JSON_VALUE(data, '$.name');
```

### 3、MERGE 增强（RETURNING）

```sql
MERGE INTO inventory AS t USING incoming AS s ON t.sku = s.sku
WHEN MATCHED THEN UPDATE SET qty = t.qty + s.qty
WHEN NOT MATCHED THEN INSERT (sku, qty) VALUES (s.sku, s.qty)
RETURNING t.sku, t.qty AS new_qty;
```

### 4、MAINTAIN 权限

```sql
GRANT MAINTAIN ON orders TO monitor_user;
-- monitor_user 可执行 VACUUM / ANALYZE / CLUSTER / REFRESH MATERIALIZED VIEW
```

### 5、Vacuum 内存优化

重新设计 Vacuum 内存结构，**最多减少 20 倍内存消耗**，提升 Vacuum 速度。

### 6、WAL 写入吞吐翻倍

WAL 写入路径优化，高并发写入场景并发事务处理能力约提升 **2 倍**。

### 7、COPY ON_ERROR

```sql
COPY bad_data FROM '/data/messy.csv'
WITH (FORMAT csv, ON_ERROR ignore, LOG_VERBOSITY verbose);
```

### 8、pg_upgrade 保留逻辑复制槽

大版本升级时不再需要删除逻辑复制槽，大幅简化升级流程。

## 十、PostgreSQL 18（2025 年 9 月）

### 1、异步 I/O（AIO）—— 最重大架构变化

读密集型场景实测最高 **3 倍性能提升**：

```ini
# postgresql.conf
io_method = io_uring   # Linux 5.1+，生产推荐
io_method = worker     # 跨平台兼容
```

### 2、虚拟生成列

```sql
CREATE TABLE orders (
  price         NUMERIC,
  qty           INT,
  total_stored  NUMERIC GENERATED ALWAYS AS (price * qty) STORED,   -- 物理存储
  total_virtual NUMERIC GENERATED ALWAYS AS (price * qty) VIRTUAL   -- 查询时计算，不占空间
);
```

### 3、uuidv7()

```sql
-- 时间戳有序 UUID，对 B-tree 索引友好（减少页分裂）
CREATE TABLE events (
  id      UUID DEFAULT uuidv7() PRIMARY KEY,
  payload JSONB
);
SELECT uuidv7();   -- '018e8f9a-1234-7abc-...' 趋势递增
SELECT uuidv4();   -- gen_random_uuid() 别名
```

### 4、B-tree 跳跃扫描（Skip Scan）

联合索引查询时，前缀列无等值条件也可利用索引：

```sql
-- 索引 (status, created_at)，17 及之前 WHERE 不含 status 则索引无法用于 created_at
-- 18：优化器通过 skip scan 遍历所有 status 值，再走 created_at
SELECT * FROM orders WHERE created_at > '2025-01-01';
```

### 5、RETURNING 访问 OLD / NEW

```sql
UPDATE accounts SET balance = balance - 100 WHERE id = 1
RETURNING OLD.balance AS before, NEW.balance AS after;

DELETE FROM logs WHERE created_at < now() - interval '30 days'
RETURNING OLD.*;
```

### 6、OR 条件索引优化

```sql
-- 18：优化器可将 OR 转为 ANY 数组查询，走索引
SELECT * FROM users WHERE status = 'active' OR status = 'pending';
-- 等价为：WHERE status = ANY(ARRAY['active','pending'])
```

### 7、OAuth 2.0 认证

```ini
# pg_hba.conf
host  mydb  appuser  0.0.0.0/0  oauth  issuer="https://auth.example.com"
```

### 8、Wire Protocol 3.2

自 PG 7.4 以来首次更新客户端/服务端通信协议，`libpq` 默认仍兼容 3.0。

### 9、数据校验和默认开启

```bash
initdb -D /data/pgdata   # 18 起默认启用 data checksums
```
